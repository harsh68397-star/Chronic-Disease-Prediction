# Chronic Disease (Diabetes) Prediction Using ML Models

Predicts the likelihood of diabetes in patients using health records, comparing
multiple classification models and addressing class imbalance and hidden missing data.

## Dataset

- Pima Indians Diabetes Dataset — 768 patient records, 8 health features + target outcome
- Source: https://www.kaggle.com/datasets/akshaydattatraykhare/diabetes-dataset

## Data Cleaning

- Discovered that Glucose, BloodPressure, SkinThickness, Insulin, and BMI used `0` as a
  placeholder for missing values rather than `NaN` — a standard null-check would have missed this
- Insulin was missing in 48.7% of records and SkinThickness in 29.6%; rather than dropping these
  high-missing columns outright, checked their correlation with the outcome first (0.30 and 0.26
  respectively) and kept both, applying median imputation
- Engineered binary "was missing" flag columns for Insulin and SkinThickness to test whether
  missingness itself was predictive — found only a weak correlation (0.04–0.05) with outcome,
  suggesting the gaps were due to inconsistent record-keeping rather than a clinical pattern

## Exploratory Data Analysis

- Target variable was moderately imbalanced: 65.1% no diabetes, 34.9% diabetes
- Glucose showed the strongest correlation with outcome (0.49), followed by BMI (0.31) and
  Insulin (0.30)
- Boxplots confirmed diabetic patients had visibly higher median Glucose (~140 vs ~107) and
  BMI (~33 vs ~30)

## Modeling

Trained and compared Logistic Regression and Random Forest, both with and without
`class_weight='balanced'` to address the class imbalance.

| Model                              | Accuracy  | Precision | Recall    | F1        |
| ---------------------------------- | --------- | --------- | --------- | --------- |
| Logistic Regression                | 69.5%     | 57.1%     | 51.9%     | 54.4%     |
| Random Forest                      | 74.0%     | 64.6%     | 57.4%     | 60.8%     |
| **Logistic Regression (balanced)** | **74.0%** | 60.9%     | **72.2%** | **66.1%** |
| Random Forest (balanced)           | 74.0%     | 61.7%     | 68.5%     | 64.9%     |
  XGBoost

**Logistic Regression with balanced class weights performed best overall**, improving recall
from 51.9% to 72.2% — catching 11 more true diabetes cases in the test set — while matching
Random Forest's accuracy. This shows that addressing class imbalance directly can matter more
than model complexity.

## Limitations

- Recall of 72% still means roughly 3 in 10 actual diabetes cases are missed — not suitable for
  real clinical use without further improvement
- Dataset size (768 records) is relatively small for a health prediction task
